## ğŸš€ Getting Started

### Prerequisites
- **Node.js 18+**
- **npm** or **pnpm**
- **Supabase account**
- **API keys** for AI models

### Installation

1. **Clone the repository**
```bash
git clone https://github.com/yourusername/mentark-quantum.git
cd mentark-quantum
```

2. **Install dependencies**
```bash
npm install
```

3. **Set up environment variables**

Create a `.env.local` file in the root directory. See [PRODUCTION_SETUP_GUIDE.md](./PRODUCTION_SETUP_GUIDE.md) for detailed instructions.

4. **Verify your setup**
```bash
npm run setup:verify
```

This will check all environment variables, Supabase connection, Pinecone index, and OAuth configuration.

5. **Set up Supabase database**

Run the migration files:
```bash
# Using Supabase CLI
supabase db reset

# Or manually execute SQL files in order:
# supabase/migrations/001_initial_schema.sql
# supabase/migrations/002_gamification_career_dna.sql
# supabase/migrations/003_teacher_admin_system.sql
# supabase/migrations/004_ml_risk_sentiment.sql
# supabase/migrations/005_multimodal_support.sql
# ... (see migrations folder for full list)
```

6. **Configure Google OAuth (Optional but Recommended)**

See [PRODUCTION_SETUP_GUIDE.md](./PRODUCTION_SETUP_GUIDE.md#step-2-oauth-configuration) for detailed OAuth setup.

7. **Generate demo data (Optional)**
```bash
npm run demo:generate
```

8. **Run the development server**
```bash
npm run dev
```

Open [http://localhost:3002](http://localhost:3002) to see the app.

## ğŸ—„ Database Setup

### Schema Overview

The database uses **PostgreSQL** with **Row Level Security (RLS)** for multi-tenancy.

**Core Tables:**
- `institutes` - Multi-tenancy base
- `users`, `students`, `teachers`, `admins` - Role-based access
- `arks`, `ark_milestones`, `ark_resources` - Learning roadmaps
- `chat_sessions`, `messages` - AI conversations
- `daily_checkins`, `emotion_timeline` - Emotional tracking
- `student_stats`, `achievements` - Gamification
- `dropout_predictions`, `career_dna` - ML outputs
- `notifications`, `parent_reports` - Communication

### Running Migrations

```bash
# Using Supabase CLI
supabase migration up

# Or manually
psql -U postgres -d your_db -f supabase/migrations/001_initial_schema.sql
```
